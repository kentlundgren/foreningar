🔒 Skydda privat data från sökmotorer

Hur förhindrar man att personnamn i datafiler (.md, .csv, .json) dyker upp i Googles sökresultat?

📋 Problemet

När man publicerar datafiler på en webbserver kan sökmotorer som Google indexera innehållet. Det innebär att om någon söker på sitt namn kan de hitta din datafil i sökresultaten – något som ofta är oönskat när det gäller medlemsregister och personuppgifter.

Denna guide beskriver flera metoder för att förhindra detta, från enklast till mest säkert.

⚠️

Redan indexerad?

Om filerna redan har indexerats av Google kan det ta veckor innan de försvinner från sökresultaten, även efter att du vidtagit åtgärder. Överväg att byta filnamn för omedelbar effekt.

🤖 Metod 1: robots.txt

Instruera sökmotorer att inte indexera Enkel Rekommenderad

En robots.txt-fil i webbplatsens rot instruerar sökmotorer att inte genomsöka eller indexera vissa filer. De flesta sökmotorer (inklusive Google) respekterar dessa instruktioner.

robots.txt (placeras i webbplatsens rot) TXT 
# Blockera alla sökmotorer från att indexera datafiler
User-agent: *

# Blockera specifika filtyper
Disallow: /*.md$
Disallow: /*.csv$
Disallow: /*.json$

# Alternativt: Blockera hela mappen med medlemsdata
Disallow: /program/Bjerred/medlemmar/*.md
Disallow: /program/Bjerred/medlemmar/*.csv
Disallow: /program/Bjerred/medlemmar/*.json
  • ✅ Enkelt att implementera
  • ✅ Fungerar på alla servertyper
  • ✅ Filerna kan fortfarande läsas av JavaScript
  • ✅ Ingen serverkonfiguration krävs
  • ⚠️ Endast en "begäran" – inte en garanti
  • ⚠️ Illasinnade aktörer kan ignorera
  • ⚠️ Filerna är fortfarande åtkomliga via direktlänk
💡

Var placeras robots.txt?

Filen måste ligga i roten av din domän, t.ex. https://kentlundgren.se/robots.txt. Den kan INTE ligga i en undermapp.

📡 Metod 2: X-Robots-Tag HTTP-header

HTTP-header som instruerar sökmotorer Medel

Genom att konfigurera webbservern att skicka en speciell HTTP-header för vissa filtyper kan du instruera sökmotorer att inte indexera dem.

För Apache-server (.htaccess):

.htaccess Apache 
# Lägg till noindex-header för datafiler
<FilesMatch "\.(md|csv|json)$">
    Header set X-Robots-Tag "noindex, nofollow"
</FilesMatch>

För nginx-server:

nginx.conf nginx 
# Lägg till noindex-header för datafiler
location ~* \.(md|csv|json)$ {
    add_header X-Robots-Tag "noindex, nofollow";
}
  • ✅ Starkare signal än robots.txt
  • ✅ Fungerar för alla filtyper
  • ✅ Filerna kan fortfarande läsas av JavaScript
  • ⚠️ Kräver serverkonfiguration
  • ⚠️ Måste veta vilken servertyp du har

🚫 Metod 3: Blockera extern åtkomst

Helt förhindra åtkomst utifrån Avancerad

Detta blockerar helt åtkomst till filerna via webbläsare. OBS: JavaScript på din sida kan då INTE heller läsa filerna!

För Apache-server (.htaccess):

.htaccess (i samma mapp som datafilerna) Apache 
# Blockera alla datafiler från extern åtkomst
<FilesMatch "\.(md|csv|json)$">
    # Returnera 403 Forbidden
    Require all denied
</FilesMatch>

# Alternativ syntax för äldre Apache-versioner:
<FilesMatch "\.(md|csv|json)$">
    Order deny,allow
    Deny from all
</FilesMatch>
🛑

Viktigt!

Om du blockerar åtkomst kommer fetch() i din JavaScript-kod att misslyckas. Trippelklick-funktionen för att visa medlemmar kommer då INTE att fungera.

  • ✅ Helt säkert – ingen kan komma åt filerna
  • ✅ Sökmotorer kan omöjligen indexera
  • ❌ JavaScript kan inte läsa filerna
  • ❌ Kräver serverkonfiguration
  • ❌ Funktionalitet på sidan slutar fungera

📁 Metod 4: Flytta till privat mapp

Organisera med en "data"-mapp Enkel

Flytta alla datafiler till en undermapp (t.ex. /data/) och blockera hela mappen i robots.txt. Detta ger bättre struktur.

Skapa mappstruktur

Skapa en mapp data/ och flytta dit alla .md, .csv, och .json-filer.

Uppdatera JavaScript-koden

Ändra sökvägarna i din kod från t.ex. medlemsdata_komplett.md till data/medlemsdata_komplett.md.

Blockera mappen i robots.txt

Lägg till Disallow: /program/Bjerred/medlemmar/data/ i din robots.txt.

  • ✅ Bättre organisation
  • ✅ Enkelt att förstå
  • ✅ Filerna fungerar fortfarande för JavaScript
  • ⚠️ Kräver uppdatering av kod
  • ⚠️ Samma begränsningar som robots.txt

🔄 Metod 5: Byt filnamn (för redan indexerade filer)

Bryt gamla länkar från sökresultat Enkel

Om filerna redan har indexerats av Google är det snabbaste sättet att få dem att "försvinna" att byta namn på filerna.

Gammalt filnamn → Nytt filnamn
medlemsdata_komplett.md → md_komplett_251226.md
medlemsdata_rå.csv → md_raw_251226.csv
medlemsdata.json → md_251226.json
✅

Tips: Använd datumstämpel

Genom att inkludera datum i filnamnet (t.ex. 251226 för 2025-12-26) blir det tydligt när datan exporterades.

  • ✅ Omedelbar effekt
  • ✅ Gamla sökresultat leder till 404
  • ✅ Enkelt att göra
  • ⚠️ Måste uppdatera alla referenser i koden
  • ⚠️ Förhindrar inte framtida indexering

🔐 Metod 6: Lösenordsskydd

Kräv autentisering för åtkomst Avancerad

Konfigurerar servern att kräva användarnamn och lösenord för att komma åt vissa filer eller mappar.

För Apache-server:

.htaccess Apache 
# Lösenordsskydd för datafiler
<FilesMatch "\.(md|csv|json)$">
    AuthType Basic
    AuthName "Skyddad data"
    AuthUserFile /path/to/.htpasswd
    Require valid-user
</FilesMatch>
⚠️

Komplex setup

Lösenordsskydd kräver att du skapar en .htpasswd-fil med krypterade lösenord och har tillgång till serverns konfiguration.

📌 Rekommenderad strategi för denna webbplats

  1. Skapa robots.txt — Lägg till blockeringsregler för .md, .csv, och .json i rotmappen.
  2. Flytta datafiler — Skapa en /data/-mapp och flytta dit alla känsliga filer.
  3. Byt filnamn — Om filerna redan är indexerade, byt namn på dem så att gamla sökresultat ger 404-fel.
  4. Uppdatera JavaScript — Se till att alla fetch()-anrop pekar på de nya filnamnen/sökvägarna.

🔍 Hur tar jag reda på vilken server jag har?

För att veta om du har Apache eller nginx, kör detta kommando i terminalen:

Terminal / PowerShell CMD 
# Windows PowerShell
Invoke-WebRequest -Uri "https://kentlundgren.se" -Method HEAD | Select-Object -ExpandProperty Headers

# Eller med curl (om installerat)
curl -I https://kentlundgren.se

Leta efter raden Server: i svaret. Den visar t.ex. Apache/2.4 eller nginx/1.18.
