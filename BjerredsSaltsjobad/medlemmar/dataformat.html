📊 Dataformat — En jämförelse

Hur lagrar man data på bästa sätt för människor och AI-modeller?

📖 Introduktion

När vi arbetar med data finns det många sätt att lagra och strukturera informationen. Valet av format påverkar hur lätt det är för människor att läsa datan, hur väl AI-modeller kan analysera den, och hur enkelt det är att importera datan till andra program.

📁 De fyra formaten

📗 Excel (.xlsx)

Microsofts kalkylprogram. Stödjer formler, formatering, diagram och flera flikar. Binärt format som kräver specialprogram för att öppnas.

  • ✓ Kraftfullt för beräkningar
  • ✓ Visuellt tilltalande
  • ✓ Välkänt för de flesta
  • ✗ Binärt format
  • ✗ Svårt för AI att läsa direkt

📄 CSV (.csv)

Comma-Separated Values. Enkel textfil där värden separeras med kommatecken. Universellt format som fungerar överallt.

  • ✓ Extremt kompakt
  • ✓ Fungerar överallt
  • ✓ Lätt för AI att parsa
  • ✗ Ingen formatering
  • ✗ Svårt att läsa för människor

📝 Markdown (.md)

Textformat med enkel formatering. Läsbart både som råtext och renderat. Populärt för dokumentation och på webben.

  • ✓ Läsbart för människor
  • ✓ Stödjer tabeller och rubriker
  • ✓ Bra för AI-analys
  • ✗ Kan bli stort med mycket data
  • ✗ Tabeller begränsade

🔷 JSON (.json)

JavaScript Object Notation. Strukturerat textformat med stöd för hierarkisk data. Standard för webb-API:er.

  • ✓ Strukturerat och hierarkiskt
  • ✓ Metadata kan inkluderas
  • ✓ Perfekt för programmering
  • ✗ Svårläst för människor
  • ✗ Större filstorlek

🤖 Hur AI-modeller hanterar data

AI-modeller som Claude, GPT-4 och Gemini läser data som text. De kan inte "öppna" en Excel-fil på samma sätt som ett kalkylprogram. Istället behöver datan konverteras till ett textbaserat format först.

Rekommendationer för AI-analys:

  • CSV — Bäst för stora datamängder, kompakt och lätt att parsa
  • Markdown — Bra balans mellan läsbarhet och struktur
  • JSON — Perfekt när metadata och struktur är viktigt
  • Kombinera! — Använd markdown för sammanfattningar + CSV för rådata

🔍 Praktiskt exempel: Hämta medlemmar för en månad

Här visar vi steg för steg hur man filtrerar ut familjemedlemmar respektive enskilda medlemmar som startade sitt medlemskap under en specifik månad (t.ex. november 2025) från en Markdown-fil.

Steg 1: Förstå Markdown-tabellens struktur

Viktiga kolumner

Medlemskap avgör om det är familj- eller enskilt medlemskap.
Startmånad (format: YYYY-MM) gör det enkelt att filtrera per månad.

Steg 2: Filtrera med JavaScript

Här är en funktion som läser Markdown-filen och filtrerar medlemmar:

filtrera_medlemmar.js JavaScript 
// Funktion för att hämta medlemmar för en specifik månad och typ
async function getMembersByMonthAndType(period, memberType) {
    // Hämta Markdown-filen
    const response = await fetch('data/md_komplett_251226.md');
    const content = await response.text();
    
    // Dela upp i rader och filtrera bort rubrik/separator
    const lines = content.split('\n');
    const dataRows = lines.filter(line => 
        line.startsWith('|') && 
        !line.includes('---') && 
        !line.includes('Användare')
    );
    
    // Parsa varje rad till ett objekt
    const members = dataRows.map(row => {
        const cols = row.split('|').filter(c => c.trim());
        return {
            nr: cols[0]?.trim(),
            namn: cols[1]?.trim(),
            medlemskap: cols[2]?.trim(),
            bokningar: cols[3]?.trim(),
            startar: cols[4]?.trim(),
            startmanad: cols[5]?.trim()
        };
    });
    
    // Definiera vilka medlemskapstyper som räknas som familj/enskild
    const familyTypes = ['Familjemedlemskap', 'Familjemedlemskap - extra'];
    const singleTypes = ['Enskilt medlemskap', 'Studentmedlemskap'];
    
    // Filtrera baserat på period och medlemstyp
    return members.filter(m => {
        const matchesPeriod = m.startmanad === period;
        const matchesType = memberType === 'familj' 
            ? familyTypes.some(t => m.medlemskap?.includes(t))
            : singleTypes.some(t => m.medlemskap?.includes(t));
        return matchesPeriod && matchesType;
    });
}

// Exempel: Hämta familjemedlemmar för november 2025
getMembersByMonthAndType('2025-11', 'familj')
    .then(members => {
        console.log(`Antal familjemedlemmar: ${members.length}`);
        members.forEach(m => console.log(m.namn));
    });

Nyckelkod från nyamedlemmar.js

nyamedlemmar.js — Formatera namn JavaScript 
/**
 * Formaterar namn: Förnamn + första bokstaven i efternamnet
 * Exempel: "Kent Lundgren" → "Kent L."
 * Detta skyddar integriteten samtidigt som listan blir läsbar.
 */
function formatName(fullName) {
    if (!fullName) return 'Okänt namn';
    
    const parts = fullName.trim().split(' ');
    
    if (parts.length === 1) {
        return parts[0];  // Bara ett namn
    }
    
    const firstName = parts[0];
    const lastName = parts[parts.length - 1];
    const lastInitial = lastName.charAt(0).toUpperCase();
    
    return `${firstName} ${lastInitial}.`;
}
nyamedlemmar.js — Hämta medlemmar för period JavaScript 
/**
 * Hämtar medlemmar för en specifik månad och typ
 * @param {string} period - Månad i format YYYY-MM (t.ex. "2025-12")
 * @param {string} memberType - "familj" eller "enskild"
 */
async function getMembersByPeriodAndType(period, memberType) {
    const allMembers = await fetchMemberData();
    
    return allMembers.filter(m => {
        const matchesPeriod = m.startmanad === period;
        const matchesType = memberType === 'familj' 
            ? isFamilyMembership(m.medlemskap)
            : isSingleMembership(m.medlemskap);
        
        return matchesPeriod && matchesType;
    });
}

💡 Rekommendationer

💡

För dagligt arbete: Excel

Om du ofta redigerar, filtrerar och beräknar – använd Excel. Det är det mest kraftfulla verktyget för interaktivt arbete med data.

🤖

För AI-analys: CSV + Markdown

Kombinera en kompakt CSV-fil för rådata med en markdown-fil som innehåller sammanfattningar och instruktioner. Detta ger AI:n kontext utan att överbelasta kontextfönstret.

⚠️

Undvik stora markdown-tabeller

En markdown-tabell med 1 000+ rader blir ca 100 KB och tar upp ~25 000 tokens. Det är ofta mer än nödvändigt. Använd istället sammanfattningar och låt AI:n fråga efter specifik data vid behov.

🔒 Skydda data från sökmotorer

När man publicerar datafiler på webben finns en risk att sökmotorer som Google indexerar innehållet. Det innebär att personnamn kan dyka upp i sökresultat – något som ofta är oönskat för medlemsregister.

🔐

Så skyddar vi medlemsdata på denna webbplats

Vi har implementerat flera skyddsåtgärder för att förhindra att personuppgifter indexeras av sökmotorer:

  • robots.txt — Instruerar sökmotorer att inte indexera .md, .csv och .json-filer
  • Separerad /data/-mapp — Alla känsliga datafiler ligger i en egen mapp som blockeras i robots.txt
  • Kryptiska filnamn — Filnamn innehåller datumstämplar (t.ex. md_komplett_251226.md) istället för beskrivande namn
