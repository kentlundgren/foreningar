Rätt repo i Cursor – parallella repon och agentisk AI

En guide till workspace-gränser, möjligheter och bästa praxis när du arbetar med flera GitHub-repon i Cursor

📋 Revision 2025 · Status v1.0 · Senast uppdaterad: 2026-06-05

Bakgrund – varför uppstår frågan?

Kent arbetar med två separata GitHub-repon för Bjerreds Saltsjöbad:

RepoInnehållGitHub Pages
kentlundgren/foreningar Inpasseringsstatistik, entréskylt, medlemmar kentlundgren.github.io/foreningar/BjerredsSaltsjobad/
kentlundgren/Bjerred-el Elenergianalys, beläggningsgrad, Firebase kentlundgren.github.io/Bjerred-el/

Cursor öppnades i mappen ClaudeCowork/BjerredsSaltsjobad/ (som är kopplad till foreningar-repot) men frågan om öppettider berör koden i Bjerred-el-repot. Det är alltså fel workspace för den uppgiften.

Vad är ett "workspace" i Cursor?

När du öppnar en mapp i Cursor (File → Open Folder) blir den mappen ditt workspace. Det har flera konsekvenser:

  • Läses Kontextfiler (CLAUDE.md, .cursor/rules/) läses automatiskt från workspace-mappen och alla dess undermappar.
  • Visas Filträdet i editorn, Source Control-panelen och git-statusen gäller workspace-repot.
  • Inte automatiskt CLAUDE.md och regler i andra repon på datorn läses inte – agenten känner inte till dem om de inte anges explicit.
Konsekvens: Om du arbetar i ett repo men frågan berör ett annat repo kan agenten svara rätt innehållsmässigt (tack vare internet-sökning och shell-åtkomst), men den manuella kontexten (regler, CLAUDE.md) från rätt projekt saknas.

Vad kan agenten göra utanför sitt workspace?

En Cursor-agent (t.ex. Claude Sonnet 4.6) har tillgång till shell-kommandon och kan därmed arbeta långt utanför workspace-mappen. Så här ser möjlighetskartan ut:

FörmågaInom workspaceUtanför workspace
Läsa filer Ja – via Read-verktyget Ja – via absolut sökväg
Skriva/redigera filer Ja – direkt Ja – via absolut sökväg
Hämta filer från GitHub – Ja – via raw.githubusercontent.com
Klona git-repo – Ja – via git clone i shell
git add / commit / push Ja – men kräver Kents godkännande Ja – men kräver Kents godkännande
Läsa CLAUDE.md / .cursor/rules automatiskt Ja – automatiskt Nej – måste anges manuellt
Visa diff i Cursor UI Ja Nej – syns inte i Source Control
Söka med Grep / Glob-verktyg Ja – effektivt Begränsat – kräver explicit sökväg

Vad hände i denna session (exempel)

Agenten befann sig i foreningar-workspace men hittade koden i Bjerred-el via tre steg:

  1. Sökte i agent-transkripter och hittade URL:en kentlundgren.github.io/Bjerred-el/
  2. Hämtade JS-filen direkt från GitHub (raw.githubusercontent.com)
  3. Klonade repot till C:\Users\kentl\Downloads\Bjerred-el\ och redigerade filerna där

Det fungerade men är inte optimalt – diff:en syns inte i Cursor UI, och CLAUDE.md för Bjerred-el lästes aldrig in automatiskt.

Varför är "rätt repo" viktigt?

1. Kontextfiler läses automatiskt

Om du öppnar Bjerred-el i Cursor läser agenten automatiskt CLAUDE.md och .cursor/rules/ för det repot. Det ger rätt regler, rätt namnkonventioner och rätt historik – utan att du behöver förklara dem.

2. Source Control-panelen fungerar

Du kan granska diff:en visuellt i Cursor innan du commitar. Ändringar i ett klonat repo i Downloads syns inte i Cursor UI – du måste starta ett separat terminalfönster för att se dem.

3. Agenten "vet" var den befinner sig

I rätt workspace hittar agenten filer via relativa sökvägar, förstår projektets struktur direkt och behöver inte gissa sig till var koden finns.

4. Risk för misstag minskar

Att redigera filer i ett klonat Downloads-repo som aldrig committades är en risk – om du glömmer att pusha försvinner arbetet. I rätt workspace är det tydligare.

Rekommenderat arbetsflöde för flera repon

Alternativ A – Separata Cursor-fönster (enklast)

Öppna ett Cursor-fönster per repo. Byt fönster när du byter uppgift.

1
Klona repot till en permanent mapp (en gång) 
cd C:\Users\kentl\OneDrive\AI\Claude
git clone https://github.com/kentlundgren/Bjerred-el.git
Välj en permanent plats – inte Downloads. Förslagsvis: C:\Users\kentl\OneDrive\AI\Claude\Bjerred-el\
2
Öppna mappen i Cursor Gå till File → Open Folder och välj C:\Users\kentl\OneDrive\AI\Claude\Bjerred-el\
3
Starta ny session med rätt startprompt 
Jag fortsätter arbetet med Bjerred-el.
Läs gärna dessa filer för kontext:
• CLAUDE.md – projektinstruktion
• README.md – projektöversikt
• fore_och_efter_ombyggnad.js – huvudlogik

Nästa uppgift: [beskriv uppgiften]

Alternativ B – Multi-root workspace (avancerat)

Cursor stöder att flera mappar läggs till i samma workspace. Då syns båda repona i filträdet och git-statusen.

  1. Öppna det primära repot (foreningar)
  2. Välj File → Add Folder to Workspace
  3. Lägg till Bjerred-el-mappen
  4. Spara workspace-filen (.code-workspace)
Obs: Multi-root workspace är kraftfullt men kan förvirra agenten om de två repona har motstridiga CLAUDE.md-regler. Använd det med försiktighet och berätta alltid för agenten vilket av repona som gäller för aktuell uppgift.

Alternativ C – Vad agenten kan göra utan klon (nödlösning)

Om du befinner dig i fel workspace och inte vill byta kan agenten:

  • Hämta filer direkt från GitHub via raw-URL
  • Klona repot temporärt (t.ex. i Downloads)
  • Göra ändringar och visa commit-kommandon som du kör manuellt

Nödlösning Det fungerar men är opraktiskt – diff syns inte i Cursor UI och risken för att glömma commit/push ökar.

Viktigaste insikten: Agenten är mäktig nog att arbeta utanför workspace – men projektminnet (CLAUDE.md, regler, context) aktiveras bara i rätt workspace. "Rätt repo" handlar därför mest om att du ska se rätt diff och slippa manuella omvägar.
